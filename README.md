# github.com/JarenDamm/jaren-mcp-go-test

A Model Context Protocol (MCP) server, generated by primus from the `mcp`
archetype (Go). It serves MCP over **streamable HTTP** — a network-reachable
endpoint that Kubernetes probes, automated tests, and real MCP clients (Claude
Code, Claude Desktop) all connect to.

This file is yours (scaffolded): edit it freely as the service grows.

## Layout

- `cmd/server/main.go` — the entrypoint (yours). Loads config + logger and
  hands off to `internal/app`.
- `internal/app/app.go` — the Foundation seam (platform-managed). Builds the
  MCP server, mounts the streamable-HTTP transport at `/mcp` alongside
  `/healthz` and `/readyz`, and owns graceful shutdown.
- `internal/app/hello.go` — a worked example tool + resource (yours). Copy this
  pattern to add your own: a file in `package app` whose `init()` calls
  `Register`.

Capabilities register tools and resources against `internal/app`; you never
edit `cmd/server/main.go` to wire them in.

## Run it

```sh
make run          # go run ./cmd/server, listens on :8080
ADDR=:9000 make run
```

The MCP endpoint is then at `http://localhost:8080/mcp`.

## Connect Claude Code

Add the server over the streamable-HTTP transport:

```sh
claude mcp add --transport http github.com/JarenDamm/jaren-mcp-go-test http://localhost:8080/mcp
```

Then start `claude` in a project and the `hello` tool is available.

## Connect Claude Desktop

Add an entry to your Claude Desktop MCP config
(`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "github.com/JarenDamm/jaren-mcp-go-test": {
      "type": "http",
      "url": "http://localhost:8080/mcp"
    }
  }
}
```

Restart Claude Desktop; the `hello` tool and the `greeting://hello` resource
appear in the MCP picker.

## Scaling caveat: SDK-default session handling

The Foundation mounts the SDK's `StreamableHTTPHandler` with its **default
in-memory session handling**. Each MCP client connection gets a server-side
session held in this process's memory. That is correct for a single replica,
but it means:

- **Sessions do not survive a restart.** A rolling deploy or pod restart drops
  every in-flight session; clients reconnect and re-initialize.
- **Sessions are not shared across replicas.** If you scale to more than one
  replica behind a load balancer, a client's follow-up requests must land on
  the same replica that holds its session. Enable sticky sessions
  (session affinity) on the load balancer, or run a single replica, until you
  move session state to a shared store.

For a stateless tool surface (tools that read everything they need from each
request), prefer the SDK's stateless mode so any replica can serve any request
without affinity. See the official Go SDK's `StreamableHTTPOptions` for the
`Stateless` toggle.
